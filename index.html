Bab 3 & 4: Metodologi Pendekatan

Alur Arsitektur Prediksi Cuaca Berbasis Random Forest

1

Pengumpulan Dataset Historis

Menarik data setebal 3-5 tahun terakhir dari stasiun cuaca satelit konvensional via Open-Meteo REST API atau BMKG. Fitur input (Suhu, Kelembaban Relative, dsb) dipetakan ke target nyata (Hujan / Cerah).

CSV FormatAPI ScrapingTime-Series
2

Data Pre-processing

Tahapan manipulasi data mentah (Imputasi Outliers, penanganan Missing Values), lalu membagi datanya menjadi 80% Data Latih (Training) dan 20% Data Uji (Testing) guna mencegah Overfitting.

3

Pemodelan Random Forest (Ensemble)

Algoritma membangun ratusan Pohon Keputusan (*Decision Trees*, N=150). Setiap pohon dipantau nilai Gini Impurity-nya dalam membelah *nodes*. Semua node final melakukan *Majority Voting* untuk memprediksi probabilitas presipitasi hujan harian.

Gini ImpurityEntropyMajority Voting
4

Evaluasi Matriks (Confusion Matrix)

Mengukur kekuatan performa tebakan cuaca algoritma dibandingkan dengan data Test 20%. Penekanannya terletak pada optimalisasi F1-Score agar prediksi cerah benar-benar terbukti cerah.

Deployment Front-End/Back-End Terintegrasi
